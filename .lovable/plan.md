## Объём

Бриф описывает 13 секций с попиксельной точностью (~1474 строки). Текущий сайт уже реализует большинство из них (Hero, PartnerBlock, ForWhomAndInside, Curriculum, Experts, News, Admission, ApplyForm, FinalCTA, Footer, AIWidget, Header). Полная пересборка — это переписать ~15 файлов с нуля, что займёт много времени и кредитов и, скорее всего, приведёт к похожему результату.

## Предлагаемый подход

Чтобы не сжигать кредиты впустую, прежде чем стирать всё:

1. **Удалить все shadcn/ui компоненты** (`src/components/ui/*`) — бриф запрещает их использовать, а сейчас они лежат мёртвым грузом.
2. **Удалить и пересоздать каждую секцию** строго по брифу:
   - `Header` — sticky с backdrop-blur, scroll-spy, mobile drawer
   - `Hero` — 55/45 split, dark feature card, counter-анимации
   - `PartnerBlock` (X5 Tech) — цитата CTO + 4 направления
   - `ForWhom` + `Inside` — две раздельные секции с альтернацией фона
   - `Curriculum` — табы с lime underline, AnimatePresence
   - `Tracks` — 4 интерактивных карточки (центральная фича)
   - `X5Touch` — вторая центральная фича из брифа
   - `Experts` — карточки с уникальным hover
   - `News` — 3 карточки с pattern-фонами
   - `Admission` + `ApplyForm` — форма с валидацией
   - `FinalCTA` + `Footer` + `AIWidget`
3. **Дизайн-токены** в `src/styles.css` ровно как в брифе (palette + типографика + spacing + shadows).
4. **Шрифты**: `@fontsource/inter` + `@fontsource/ibm-plex-mono` (не CDN).
5. **Чек-листы из брифа**: альтернация фонов, разные дивайдеры, отсутствие horizontal scroll на 320–1440px.

- **(a)** Действительно пересобирать ВСЁ с нуля, или
- **(b)** Указать 2–3 конкретные секции, которые сильнее всего расходятся с брифом (например «Hero выглядит не так» / «нет секции Tracks» / «шрифты не загружаются»), и я аккуратно их переделаю?

Вариант (b) обычно даёт лучший результат на тех же кредитах.
